# Chronicle Design System

> A unified design language for Chronicle — the infrastructure layer
> for agent behavior.

Serif headlines for authority, mono for time and topics, sans for
reading. A restrained dark surface with one signature: the
**ember → sage → ember** light-source gradient.

## Three principles

1. **Serif for judgement. Mono for evidence.**
   Every claim about behavior lands in Kalice; every piece of evidence
   lands in Geist Mono.
2. **One hot surface. Everything else rests.**
   The ember gradient is a signal, not decoration. It marks the living
   scenario, the selected event, the active version — and nothing else.
3. **Streams are colored. Systems are quiet.**
   Event streams each own a hue (teal for intercom, amber for shopify,
   green for stripe). System chrome stays neutral so your eye tracks
   causality.

## Navigating this book

- **Foundations** — tokens: color, typography, spacing, motion,
  gradient/glass recipes.
- **Primitives** — Button, Input, Badge, Panel, StatusDot, etc.
- **Typography** — Display / Body / Mono components.
- **Surfaces** — the brand light-source + glass treatments.
- **Product** — the event stream, replay suite, and the pieces they
  compose from.
- **Layout** — app shells (topbar + sidebar + detail + footer).
- **Templates** — full-page recreations of the handoff's pages 05–07.

Every story has a **theme** switch in the toolbar — dark is the brand
default, light is the warm-bone alt theme.
